APRIL 7, 2026

Agent Identity Architecture — Five Patterns, A2A Protocol, Agent Sprawl

TypeChangePage(s) Affected
🆕 New Agent's User Account — fifth authentication pattern — an agent provisioned with a full human user account (mailbox, calendar, Teams membership). Highest risk pattern: compromised agent is indistinguishable from a human user. Added to the authentication patterns table. identity.html
🆕 New Agent sprawl — named concept and lifecycle risk section — Microsoft formally defines agent sprawl as uncontrolled expansion of agents without visibility, management, or lifecycle controls. Added to identity page and gaps page with consequences and mitigations. identity.html, gaps.html
🆕 New A2A (Agent-to-Agent) protocol — emerging standard for authenticated inter-agent communication, supported by Entra Agent ID alongside MCP. Added to identity page and MCP page with comparison table and risk callout. identity.html, mcp.html
🆕 New Agent-to-agent propagation — new threat scenario (Scenario 7) — compromised orchestration agent propagates compromise to sub-agents across the entire agent chain. Full attack chain with controls and A2A gap note. threats.html
🆕 New Microsoft Managed Policies for agents — new significant gap — automatic baseline CA policies that block high-risk agents. Many organisations unaware of or not using these. Added to gaps page. gaps.html

MARCH 30, 2026

Agent Authentication Patterns + CA for Agents Correction

TypeChangePage(s) Affected
⚠️ Correction Conditional Access for Agents does NOT apply to Copilot Studio agents — corrected on both the identity page and product map card. CA for Agents only triggers during modern Agent ID authentication (OAuth 2.0), used by Security Copilot and AI Foundry. Copilot Studio agents use OBO, maker credentials, or service principal — none of which trigger CA for Agents. Field-validated by Derk van der Woude (March 2026). identity.html, product-map.html
🆕 New Four Copilot Studio authentication patterns table — new section on identity page covering all four patterns: End User Credentials (OBO), Maker-Provided Credentials, App Registration Delegated, App Registration Application Permissions. Includes risk rating and detection method for each. identity.html
🆕 New Precise maker credentials KQL — upgraded Playbook 01 Step 4 with Derk's field-validated query that checks both AgentToolsDetails and AgentTopicsDetails for maker mode connections. More precise than the previous agent-level auth type check. playbooks.html
🆕 New App Registration Graph API detection KQL — new Playbook 01 Step 4b detects agents using HTTP Request actions to graph.microsoft.com or management.azure.com, identifying potential application permission agents (very high risk — tenant-wide access). playbooks.html
🆕 New Change-detection KQL for auth type downgrade — added to Playbook 01 Step 1. Detects when a published agent's authentication is changed to None — designed to be saved as a Sentinel Analytics Rule for real-time alerting. Sourced from Derk's AI Agent Inventory blog (November 2025). playbooks.html
🆕 New Any user can change another agent's auth type — new significant gap — by design in Copilot Studio, any tenant user can downgrade another agent's authentication to No Authentication, even without being the owner. Added to Significant Gaps with interim mitigations. gaps.html
🆕 New Community Queries tip added to Playbook 01 — Defender Advanced Hunting has a dedicated AI Agents section with queries from the Microsoft Product Group. Callout added to Playbook 01 checklist. playbooks.html
📌 Source

Derk van der Woude — "Your Copilot Studio agent is acting as someone, do you know who?" (March 2026)

MARCH 28, 2026

Purview DLP External Web Search Blocking

TypeChangePage(s) Affected
🆕 New Purview DLP external web search blocking (Coming June/July 2026) — new DLP policy option to prevent Copilot from sending prompts containing selected Sensitive Information Types (SITs) to external web search. When triggered, Copilot continues responding using internal Microsoft Graph data only. Alerts in DLP Alerts and Activity Explorer under DSPM for AI. GA June/July 2026, opt-in required. product-map.html, gaps.html
⚠️ Correction Purview DLP SIT blocking description corrected — initial entry incorrectly stated: (a) the feature was "coming June/July 2026" — it is already in Preview, June/July is the GA target; (b) Copilot "continues responding using internal Graph sources" when triggered — it does not respond at all; (c) only external web search was blocked — both internal and external searches are blocked. Also added: files uploaded directly into prompts are not scanned by DLP (only typed text); the two DLP conditions (SITs and sensitivity labels) cannot be in the same rule. product-map.html, gaps.html
📌 Sources

Microsoft Purview product announcement (March 2026) · learn.microsoft.com — DLP for M365 Copilot (official docs, updated Feb 2026)
