What Agent 365 Is

The enterprise layer above your agent logic

Agent 365 is not an agent builder or hosting platform. It is a management and security layer that sits above whatever platform your agents run on — Copilot Studio, Microsoft Foundry, LangChain, OpenAI Agents SDK, or anything else. Once an agent is onboarded to Agent 365, it gains enterprise-grade controls it didn't have before.

🪪
Entra-backed Agent Identity
Each agent gets its own identity in Microsoft Entra ID — including a dedicated mailbox and user resources for secure authentication. Enables Conditional Access, ID Protection, and lifecycle governance per agent.
🔌
Governed MCP Tool Access
Agents invoke MCP servers under admin control via the Agent Tooling Gateway (ATG). Tool invocations are evaluated by Defender before execution — unsafe actions blocked in real time. No open-ended permissions.
📊
OpenTelemetry Observability
Agent interactions, inference events, and tool usage are instrumented automatically via the Agent 365 SDK. All events routed to Microsoft 365 audit logs — visible in Defender Advanced Hunting via the AIAgentsInfo table.
📋
Blueprint-Based Governance
Each agent operates within an IT-approved blueprint defining capabilities, required MCP accesses, security constraints, audit requirements, and linked DLP or external access policies. Consistent configuration at scale.
🔔
M365 Notifications
Agents can participate in M365 apps like a human participant — via @mentions in Teams, comments in Word, and Outlook notifications. Enables agents to surface work to users in their existing workflows.
🛡️
Defender Integration
Agents registered with Agent 365 appear in the AIAgentsInfo Advanced Hunting table (RegistrySource == "A365"). Near-real-time threat detection, incident alerts, and investigation via Defender portal → Settings → Security for AI.

📌 What "platform-agnostic" means in practice

You add the Agent 365 SDK to your agent code. Once instrumented, the agent registers with Agent 365 regardless of where it runs — Azure, AWS, GCP, or your own infrastructure. The SDK handles the Entra identity registration, OpenTelemetry event emission, and ATG integration automatically. You don't rewrite your agent; you add a governance layer around it.

Access & Preview

Frontier programme — current preview access

Agent 365 is GA on May 1, 2026. Before that date — and for some advanced preview features — access is via the Microsoft Frontier programme.

1
Enrol in Frontier — go to adoption.microsoft.com/copilot/frontier-program and request access. Frontier gives early access to Agent 365, Entra Agent ID, and related preview capabilities.
2
Enable in Power Platform admin center — for Copilot Studio agents: Power Platform admin → Copilot → Settings → Entra Agent Identity for Copilot Studio → On. This makes new Copilot Studio agents Modern agents automatically.
3
Connect Defender — Defender portal → Settings → Security for AI agents → enable and connect your Agent 365 tenant. Agents registered with Agent 365 start appearing in AIAgentsInfo.
4
Instrument custom agents — add the Agent 365 SDK to agents not on Copilot Studio. This gives them Entra identity, OpenTelemetry observability, and ATG real-time protection. Available on PyPI and npm.

Key KQL — Agent 365 Agents

Essential Advanced Hunting queries

Use RegistrySource == "A365" to target Agent 365-registered agents specifically. See Playbook 01 Step 8 for the full query set.

// All A365 registered agents
AIAgentsInfo
| where RegistrySource == "A365"
| summarize arg_max(Timestamp, *) by AIAgentId
| where AgentStatus != "Deleted"
| project AIAgentId, AIAgentName, AgentStatus, IsBlocked, AIModel, Instructions

// Agents with no instructions — prompt injection risk
AIAgentsInfo
| where RegistrySource == "A365"
| summarize arg_max(Timestamp, *) by AIAgentId
| where isempty(Instructions) or Instructions == "N/A"
| project AIAgentId, AIAgentName, Instructions

// Agents with MCP tools — expanded attack surface
AIAgentsInfo
| where RegistrySource == "A365"
| summarize arg_max(Timestamp, *) by AIAgentId
| where isnotempty(AgentActionTriggers)
| extend Triggers = parse_json(AgentActionTriggers)
| mv-expand Trigger = Triggers
| where Trigger.type == "RemoteMCPServer"
| project AIAgentId, AIAgentName, Trigger.type
